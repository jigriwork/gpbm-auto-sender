# Public Homepage

The `/` route is now a public SaaS landing page instead of redirecting directly to `/dashboard`.

## Positioning

Core message:

> Send every bill on WhatsApp automatically, without changing your billing software.

The homepage positions GPBM Auto Sender for Indian retail businesses that already save or export PDF bills from billing software such as Logic, Marg, Tally, or custom systems.

## Sections built

- Header with product name, section anchors, login CTA, and how-it-works CTA.
- Hero with product positioning and a dashboard-style bill flow visual.
- Problem section for expensive integrations, cashier workload, customer expectations, and tracking needs.
- Solution/features section covering folder watcher, PDF extraction, provider selection, dashboard tracking, duplicate protection, and retry system.
- How it works section from saved PDF to dashboard tracking.
- Supported software section with Generic PDF Folder Watcher, Logic, Marg, Tally, and Custom PDF.
- WhatsApp provider section with MSG91, Custom API, and coming-soon providers.
- Multi-store SaaS section covering businesses, stores, credentials, dashboards, and super-admin support.
- Dashboard preview metrics.
- Pricing placeholders for Starter, Growth, and Scale.
- FAQ.
- Footer with login and security note.

## Current limits

- Pricing is placeholder only.
- No payment or subscription checkout is implemented.
- Marketing routes like `/features` and `/pricing` are section anchors, not separate pages.
- Public signup, lead capture, and contact/demo request flows are not built yet.
- Homepage uses a coded product mock visual rather than real screenshots.
